# Flowgence 기능 명세서 (비즈니스 관점)

> **Flowgence**는 SI(시스템 통합) 프로젝트의 초기 단계를 AI로 자동화하여,
> 고객은 쉽게 프로젝트를 의뢰하고, SI 업체는 사전 분석에 드는 비용과 시간을 대폭 줄이는 서비스입니다.

## 1. 서비스가 해결하는 문제

### 기존 SI 프로젝트의 초기 단계 문제점

| 문제 | 상세 |
|------|------|
| **소통 비용** | 고객이 원하는 것을 정확히 표현하기 어렵고, SI 업체가 이해하기까지 수차례 미팅 필요 |
| **요구사항 누락** | 구두/메일 기반 소통으로 요구사항이 빠지거나 모호하게 전달됨 |
| **견적 불투명** | 고객은 비용이 왜 그렇게 나오는지 모르고, SI 업체는 산출 근거를 일일이 설명해야 함 |
| **초기 비용** | 본 계약 전 요구사항 분석과 견적 산출에 인력과 시간이 소모됨 |
| **시각화 부재** | 고객이 최종 결과물을 상상하기 어려워 기대치 불일치 발생 |

### Flowgence의 해결 방식

**AI가 고객과 대화하면서 요구사항을 추출하고, 자동으로 견적과 와이어프레임까지 생성**

- 고객: 전문 지식 없이도 AI 채팅만으로 프로젝트 의뢰 가능
- SI 업체: 정제된 요구사항 + 견적 + 와이어프레임을 즉시 확보

## 2. 핵심 기능: AI 기반 프로젝트 생성 워크플로우

> 경로: `/` (메인 페이지)
>
> Flowgence의 **가장 핵심적인 기능**으로, 고객이 프로젝트를 처음 의뢰할 때 사용하는 6단계 위자드입니다.

---

### Step 1. 서비스 유형 선택

**비즈니스 목적:** 고객의 도메인에 맞는 AI 분석 컨텍스트를 사전에 설정하여, 더 정확한 요구사항 추출과 견적 산출을 가능하게 합니다.

**제공 유형:**
- 음식 배달 서비스
- 부동산 플랫폼
- 업무 관리 도구
- 온라인 교육 플랫폼
- 이커머스

**이 기능이 있는 이유:**
- 도메인별로 필요한 기능 세트가 다름 (배달앱 → 실시간 위치추적, 교육 → LMS 기능 등)
- AI가 도메인 지식을 바탕으로 고객이 미처 생각하지 못한 요구사항까지 제안 가능
- 고객 입장에서 "빈 종이에서 시작"하는 부담을 줄여 진입 장벽을 낮춤

---

### Step 2. 파일 업로드 및 설명 입력

**비즈니스 목적:** 고객이 이미 보유한 기획 자료를 활용하여 AI 분석의 정확도를 높이고, 고객의 수고를 줄입니다.

**기능 상세:**
- 드래그 앤 드롭 파일 업로드 (PDF, 문서, 이미지)
- 업로드된 파일에서 텍스트 자동 추출 (`extractContentFromFiles`)
- 파일 크기/타입 검증
- 프로젝트에 대한 자유 설명 입력

**이 기능이 있는 이유:**
- 대부분의 고객은 이미 간단한 기획서, 벤치마킹 자료, 화면 캡처 등을 보유
- 이 자료를 AI가 직접 읽고 분석하면, 고객이 모든 것을 처음부터 설명할 필요 없음
- 기존 SI 프로세스에서 "자료 전달 → 담당자 검토 → 질문 → 답변" 사이클을 한 번에 단축

---

### Step 3. AI 채팅 기반 요구사항 수집

**비즈니스 목적:** 전문 PM/SA가 하던 요구사항 인터뷰를 AI가 대신 수행하여, 24시간 언제든 고객이 프로젝트를 의뢰할 수 있게 합니다.

**기능 상세:**
- AI와 실시간 대화형 인터페이스
- 업로드된 파일과 서비스 유형을 컨텍스트로 활용
- 채팅 히스토리 저장 및 이어하기 지원
- 메시지 버블, 타이핑 인디케이터 등 자연스러운 대화 UX

**이 기능이 있는 이유:**
- 고객은 "정형화된 폼"보다 "대화"로 설명하는 것이 자연스러움
- AI가 맥락을 이해하고 꼬리 질문을 하면서, 고객 스스로 인지하지 못한 요구사항까지 도출
- 기존: 미팅 잡기 → 대면 인터뷰 → 회의록 정리 (1~2주) → AI 채팅으로 즉시 완료
- 채팅 기록이 남아 나중에 "고객이 뭐라고 했는지" 분쟁 소지 방지

---

### Step 4. 프로젝트 개요 자동 생성

**비즈니스 목적:** 채팅 내용을 구조화된 프로젝트 개요로 변환하여, 고객과 SI 업체가 같은 그림을 보고 있는지 확인합니다.

**생성 항목:**
- 프로젝트 핵심 요소 정리
- 주요 기능 목록
- 타겟 사용자 정의
- 기술적 고려사항

**이 기능이 있는 이유:**
- 채팅은 비정형 → 이를 정형화된 문서로 변환해야 다음 단계 진행 가능
- "고객이 말한 것"과 "AI가 이해한 것"을 대조하여 오해를 조기에 발견
- SI 업체 내부에서 프로젝트 검토 시 표준화된 포맷으로 빠르게 파악 가능

---

### Step 5. 요구사항 추출 및 검증

**비즈니스 목적:** SI 프로젝트의 가장 중요한 산출물인 "요구사항 명세서"를 AI가 자동으로 생성하고, 품질을 검증합니다.

**기능 상세:**

| 기능 | 목적 |
|------|------|
| AI 자동 추출 | 채팅 + 파일에서 요구사항을 카테고리별로 분류하여 추출 |
| 카테고리 분류 | 기능/비기능/기술/비즈니스/UI·UX/보안/성능/통합 — 누락 영역 식별 |
| 우선순위 설정 | critical/high/medium/low — 개발 순서와 MVP 범위 결정 근거 |
| AI 검증 | 추출된 요구사항의 완전성, 일관성, 실현 가능성 자동 검토 |
| AI 추천 | 고객이 놓친 요구사항을 도메인 지식 기반으로 제안 |
| 수동 편집 | 고객이 직접 요구사항을 추가/수정/삭제하여 최종 확정 |
| 인수 조건 | 각 요구사항별 "완료" 기준 자동 생성 → 개발 완료 후 검수 기준 |

**이 기능이 있는 이유:**
- 요구사항 누락은 SI 프로젝트 실패의 1순위 원인
- 경험 많은 PM이 아니면 카테고리별 균형 잡힌 요구사항 정리가 어려움
- AI 검증으로 "이 요구사항은 모호합니다", "보안 관련 요구사항이 없습니다" 등 피드백 제공
- 고객이 직접 수정할 수 있어 "AI가 만들고 고객이 확인"하는 협업 구조

---

### Step 6. 최종 확인 및 제출

**비즈니스 목적:** 모든 단계의 결과물을 한눈에 검토하고, 고객이 최종 승인하여 프로젝트를 공식적으로 시작합니다.

**기능 상세:**
- 프로젝트 개요 + 요구사항 + 견적 통합 확인 화면
- 최종 확인 모달 (되돌릴 수 없음을 명시)
- Supabase에 프로젝트 데이터 영구 저장

**이 기능이 있는 이유:**
- 고객의 명시적 승인 없이 프로젝트가 진행되면 분쟁 소지
- 모든 정보를 한 곳에서 최종 검토하여 "나중에 몰랐다"는 상황 방지
- 제출 시점이 프로젝트 라이프사이클의 공식 시작점

---

### 워크플로우 보조 기능

| 기능 | 비즈니스 목적 |
|------|--------------|
| **세션 자동 저장** | 고객이 작성 중 이탈해도 데이터 유실 없음 → 이탈률 감소 |
| **프로젝트 이어하기** | 한 번에 완료하지 않아도 됨 → 고객 부담 경감 |
| **진행률 바** | 현재 어디까지 왔는지 시각화 → 완주율 향상 |
| **모바일 반응형** | PC 없이도 모바일에서 의뢰 가능 → 접근성 확대 |
| **로그인 유도 모달** | 비로그인 상태에서 진행 시 적절한 시점에 로그인 요청 → 데이터 연결 보장 |

## 3. 고객 관리: 마이페이지

> 경로: `/mypage/*`
>
> 고객이 자신의 프로젝트 현황을 한눈에 파악하고, 필요한 산출물을 확보하는 공간입니다.

---

### 3-1. 대시보드 (`/mypage`)

**비즈니스 목적:** 고객에게 자신의 프로젝트 전체 현황을 한눈에 제공하여, 별도 문의 없이도 상태를 파악할 수 있게 합니다.

**기능 상세:**

| 기능 | 비즈니스 목적 |
|------|--------------|
| **통계 카드** (진행중/완료/총견적/승인대기) | 고객이 현재 포트폴리오 전체 상태를 즉시 파악 |
| **최근 프로젝트 그리드** (최대 6개) | 가장 관심 있는 프로젝트에 빠르게 접근 |
| **상태 필터/정렬** | 여러 프로젝트를 운영하는 고객이 원하는 정보만 빠르게 조회 |
| **견적서 PDF 다운로드** | 내부 보고, 예산 승인, 타 업체 비교견적 등에 즉시 활용 |
| **프로젝트 이어하기** | 미완료 프로젝트를 마이페이지에서 바로 재개 |

**이 기능이 있는 이유:**
- 고객이 "내 프로젝트 어떻게 됐어요?" 문의를 하지 않아도 되는 셀프서비스 환경
- SI 업체의 CS 부담 감소
- 견적서를 즉시 다운로드할 수 있어 고객사 내부 의사결정 속도 향상

---

### 3-2. 진행중인 프로젝트 (`/mypage/in-progress`)

**비즈니스 목적:** 현재 활성 상태인 프로젝트만 모아보여, 고객이 "지금 해야 할 일"에 집중할 수 있게 합니다.

**기능 상세:**

| 기능 | 비즈니스 목적 |
|------|--------------|
| **세부 상태 필터** (요구사항 검토/추출/견적/계약 등) | 어떤 단계에서 멈춰있는지 파악 → 고객 액션 유도 |
| **"작성 이어하기" 버튼** | 중단된 지점에서 바로 재개 → 완료율 향상 |
| **빈 상태 안내** | 진행중 프로젝트가 없으면 새 프로젝트 생성 유도 → 전환 촉진 |

**이 기능이 있는 이유:**
- 프로젝트가 어느 단계에서 멈췄는지 고객이 직접 확인
- "이어하기" 기능으로 중도 이탈 프로젝트 복구 → 비즈니스 기회 손실 방지

---

### 3-3. 완료된 프로젝트 (`/mypage/completed`)

**비즈니스 목적:** 완료된 프로젝트의 산출물을 영구 보관하고, 언제든 다시 참조/다운로드할 수 있게 합니다.

**기능 상세:**

| 기능 | 비즈니스 목적 |
|------|--------------|
| **견적서 PDF 다운로드** | 상세 견적서 생성 (단계별 비용, 결제일정, 팀구성, 요구사항 요약) |
| **요구사항 수/견적 금액 표시** | 프로젝트 규모를 한눈에 비교 |
| **상세보기** | 프로젝트 전체 내용을 다시 확인 |

**이 기능이 있는 이유:**
- 고객사 내부에서 과거 프로젝트를 레퍼런스로 활용 (유사 프로젝트 비교)
- 견적서를 분실해도 언제든 재다운로드 가능
- 감사/회계 목적으로 프로젝트 이력 보관 필요

---

## 4. 외부 협업: Notion 연동

> 경로: `/mypage/settings`
>
> 고객이 이미 사용 중인 협업 도구(Notion)와 Flowgence를 연결합니다.

---

### 기능 상세

| 기능 | 비즈니스 목적 |
|------|--------------|
| **Notion OAuth 연결** | 고객의 기존 Notion 워크스페이스에 안전하게 연동 |
| **요구사항 Notion 공유** | 추출된 요구사항을 Notion 데이터베이스에 자동 등록 |
| **견적서 Notion 공유** | 견적서를 Notion 페이지로 자동 생성 |
| **연결 해제** | 고객이 원할 때 언제든 연동 해제 가능 (데이터 주권) |
| **데이터베이스 ID 설정** | 고객이 원하는 Notion 데이터베이스를 직접 지정 |

### 이 기능이 있는 이유

- **고객 워크플로우 통합:** 많은 IT 기업이 Notion을 프로젝트 관리에 사용 → Flowgence 산출물을 고객의 기존 워크플로우에 자연스럽게 편입
- **수동 복사 제거:** 요구사항을 하나씩 Notion에 옮기는 수작업 제거
- **팀 공유 용이:** Notion에 공유되면 고객사 내부 팀원들이 바로 검토/코멘트 가능
- **Lock-in 방지:** 데이터를 Flowgence 안에만 가두지 않고, 고객이 원하는 곳으로 내보낼 수 있음

---

## 5. 운영 관리: 관리자 시스템

> 경로: `/admin/*`
>
> SI 업체(Flowgence 운영팀)가 전체 서비스를 모니터링하고 관리하는 내부 도구입니다.

---

### 5-1. 관리자 대시보드 (`/admin`)

**비즈니스 목적:** 서비스 전체 현황을 한눈에 파악하여, 운영 의사결정을 빠르게 내립니다.

| 기능 | 비즈니스 목적 |
|------|--------------|
| **전체 통계** (프로젝트 수/완료/견적합계/대기) | 서비스 KPI를 실시간 모니터링 |
| **최근 프로젝트 목록** | 신규 유입 프로젝트를 빠르게 확인하여 대응 |
| **승인 대기 목록 + 대기 일수** | 오래 방치된 프로젝트를 식별 → SLA 위반 방지 |
| **프로젝트 상세 모달** | 개별 프로젝트 심층 검토 (개요/견적/요구사항) |
| **검색** | 특정 프로젝트를 빠르게 찾기 |

**이 기능이 있는 이유:**
- SI 업체가 들어오는 프로젝트를 놓치지 않고 적시에 대응
- "승인 대기 N일"로 고객 응대 지연 경고 → 고객 만족도 유지
- 경영진이 매출 파이프라인(총 견적 금액)을 실시간 확인

---

### 5-2. 사용자 관리 (`/admin/users`)

**비즈니스 목적:** 고객 정보를 관리하고, 서비스 이용 패턴을 파악합니다.

| 기능 | 비즈니스 목적 |
|------|--------------|
| **사용자 목록 + 검색** | 특정 고객을 빠르게 찾기 |
| **회사별 분류** | B2B 고객사 관리 |
| **프로젝트 수 표시** | 고객별 이용 빈도 파악 → VIP 고객 식별 |
| **역할 관리** (admin/user/client) | 권한 분리로 보안 유지 |
| **가입일/상태** | 고객 라이프사이클 관리 |
| **페이지네이션** | 대규모 고객 기반에서도 빠른 탐색 |

**이 기능이 있는 이유:**
- 고객 데이터는 B2B SaaS의 핵심 자산
- 많이 이용하는 고객에게 우선 대응, 이탈 징후 고객에게 리텐션 활동
- 역할별 접근제어로 민감한 데이터 보호

---

### 5-3. 프로젝트 모니터링 (`/admin/projects`)

**비즈니스 목적:** 전체 프로젝트의 진행 상황을 실시간으로 추적하여, 병목 구간을 식별하고 운영 효율을 높입니다.

| 기능 | 비즈니스 목적 |
|------|--------------|
| **상태 필터** | 특정 단계에 몰린 프로젝트 식별 → 프로세스 병목 발견 |
| **프로젝트명/사용자 검색** | CS 문의 시 해당 프로젝트 즉시 조회 |
| **견적 금액/요구사항 수** | 프로젝트 규모별 분류 → 리소스 배분 판단 |
| **상세 모달** (개요/견적/요구사항) | 영업팀이 고객 미팅 전 프로젝트 전체를 빠르게 파악 |
| **페이지네이션** | 대량 프로젝트에서도 성능 유지 |

**이 기능이 있는 이유:**
- SI 업체 운영팀이 "어떤 프로젝트가 어디에 있는지" 항상 파악할 수 있어야 함
- 상태별 필터로 "견적 대기 중인 프로젝트가 20개" 같은 인사이트 즉시 확보
- 프로젝트별 상세 검토로 실제 계약 진행 여부를 판단

## 6. 인증 및 접근제어

### 6-1. 로그인 (`/auth/login`)

**비즈니스 목적:** 고객 식별 및 데이터 보호

| 기능 | 비즈니스 목적 |
|------|--------------|
| **이메일/비밀번호 인증** | 기본적인 고객 식별 |
| **리다이렉트 지원** (`redirectStep`) | 로그인 전 작업 중이던 단계로 자동 복귀 → UX 연속성 |
| **에러 메시지** | 로그인 실패 시 명확한 안내 → CS 문의 감소 |

### 6-2. 회원가입 (`/auth/signup`)

**비즈니스 목적:** 고객 온보딩 및 B2B 정보 수집

| 기능 | 비즈니스 목적 |
|------|--------------|
| **이메일/비밀번호** | 계정 생성 |
| **이름/회사명** | B2B 고객 프로필 구축 → 영업/CS에 활용 |
| **이메일 인증** | 유효한 이메일 확보 → 커뮤니케이션 채널 보장 |
| **비밀번호 검증** (6자 이상, 확인 일치) | 보안 최소 기준 충족 |

### 6-3. 역할 기반 접근제어 (RBAC)

| 역할 | 접근 범위 | 비즈니스 목적 |
|------|-----------|--------------|
| **admin** | 관리자 대시보드, 사용자 관리, 프로젝트 모니터링 | SI 업체 운영팀 전용 |
| **user** | 프로젝트 생성, 마이페이지 | 일반 고객 |
| **client** | 제한된 조회 | 고객사 내부 검토자 (향후 확장용) |

**이 기능이 있는 이유:**
- 고객은 자기 프로젝트만, 관리자는 전체를 봐야 하는 명확한 권한 분리
- 데이터 유출/오조작 방지
- B2B 서비스에서 역할별 접근제어는 기본 보안 요건

## 7. 고객 접점: 문의하기

> 경로: `/contact`

**비즈니스 목적:** 서비스에 대한 질문이나 기술적 문제를 쉽게 전달할 수 있는 창구를 제공합니다.

| 기능 | 비즈니스 목적 |
|------|--------------|
| **문의 폼** (이름/이메일/메시지) | 고객이 별도 이메일 클라이언트 없이 바로 문의 |
| **연락처 정보 표시** | 이메일, 주소 등 공식 연락처 안내 |
| **폼 검증** (Zod) | 불완전한 문의 방지 → CS 효율 향상 |
| **성공/에러 피드백** | 문의가 접수되었는지 고객에게 즉시 확인 |

**이 기능이 있는 이유:**
- AI 자동화가 모든 상황을 커버하지 못함 → 인간 대응 채널 필수
- 잠재 고객(미가입자)도 문의 가능 → 리드 획보
- 문의 내용은 이메일(Nodemailer)로 운영팀에 전달 → 놓침 방지

## 8. 미구현 페이지 (향후 확장)

현재 플레이스홀더(스텁)로만 존재하는 페이지들의 **예상 비즈니스 목적:**

| 경로 | 예상 목적 |
|------|-----------|
| `/project` | 전체 프로젝트 목록 — 마이페이지와 별도로 프로젝트 중심 네비게이션 |
| `/project/new` | 독립 페이지형 프로젝트 생성 — 현재 `/` 위자드의 분리 버전 |
| `/project/new/requirements` | 요구사항 전용 페이지 — 대규모 요구사항 편집 시 더 넓은 화면 |
| `/project/new/confirmation` | 확인 전용 페이지 |
| `/project/new/completion` | 완료 안내 페이지 — 다음 단계 안내 및 CTA |
| `/project/[id]/overview` | 개별 프로젝트 개요 — 프로젝트별 독립 대시보드 |
| `/project/[id]/requirements` | 프로젝트별 요구사항 상세 — 계약 후 요구사항 변경 관리 |
| `/project/[id]/estimation` | 프로젝트별 견적 상세 — 견적 비교/수정 |
| `/project/[id]/contract` | 계약 검토 — 전자계약, 서명 기능 (예상) |

**향후 확장 방향:**
- 현재는 모든 핵심 로직이 `/`(메인 위자드)에 집중
- 프로젝트가 복잡해지면 각 단계를 독립 페이지로 분리하여 더 깊은 기능 제공
- 특히 `/project/[id]/contract`는 견적 승인 → 전자계약까지의 플로우를 완성하는 핵심 확장 포인트

## 9. 전체 비즈니스 플로우

```
[고객 여정]

   접속 (/) → 서비스 유형 선택 → 파일 업로드
        ↓
   AI 채팅으로 요구사항 설명
        ↓
   AI가 프로젝트 개요 자동 생성 ← "고객과 같은 그림을 보고 있는가?"
        ↓
   AI가 요구사항 자동 추출 + 검증 ← "빠진 것은 없는가?"
        ↓
   고객이 검토/수정/승인 ← "고객의 명시적 동의"
        ↓
   최종 제출 → 프로젝트 생성 완료
        ↓
   마이페이지에서 관리
   ├── 진행 상태 확인
   ├── 견적서 PDF 다운로드 → 내부 보고
   └── Notion으로 공유 → 팀 협업


[운영팀 여정]

   관리자 대시보드 → 신규 프로젝트 확인
        ↓
   프로젝트 상세 검토 (요구사항 + 견적)
        ↓
   고객 컨택 → 계약 진행
        ↓
   사용자 관리 → 고객 관계 유지
```

### 비즈니스 가치 요약

| 이해관계자 | Flowgence가 제공하는 가치 |
|-----------|-------------------------|
| **고객** | 전문 지식 없이 AI 대화만으로 SI 프로젝트 의뢰 가능. 24시간, 즉시, 무료로 요구사항 분석과 견적 확보 |
| **SI 업체 영업팀** | 사전 분석된 요구사항 + 견적으로 고객 미팅 준비 시간 대폭 단축 |
| **SI 업체 PM/SA** | AI가 1차 분석한 구조화된 요구사항을 바탕으로, 정제 작업에만 집중 |
| **SI 업체 경영진** | 프로젝트 파이프라인(건수, 견적 총액)을 실시간 모니터링. 승인 대기 SLA 관리 |
