# Flowgence 페이지별 기능 가이드

---

## 1. 공통 헤더

> 모든 페이지 상단에 표시되는 네비게이션 바

### 비로그인 상태

| 영역 | 내용 |
|------|------|
| 좌측 | Flowgence 로고 (클릭 시 홈으로 이동) |
| 우측 | `로그인` · `회원가입` · `문의하기` 링크 |

### 로그인 상태

| 영역 | 내용 |
|------|------|
| 좌측 | Flowgence 로고 |
| 우측 | `안녕하세요, OOO님!` 인사말 · `문의하기` · `마이페이지` · `로그아웃` |
| 우측 (admin) | 위 항목 + `관리자 페이지` 추가 |

- 모바일에서는 햄버거 메뉴로 전환

## 2. 홈 — 메인 프로젝트 생성 위자드 (`/`)

> Flowgence의 핵심 페이지. 고객이 AI와 대화하며 프로젝트를 생성하는 멀티스텝 위자드.

### Step 1. 서비스 유형 선택 + 파일 업로드

**화면 구성:**
- 상단: 진행률 바 (ProgressBar)
- 서비스 유형 선택 버튼 그룹 (음식 배달, 부동산, 업무 관리, 온라인 교육, 이커머스)
- 파일 업로드 영역 (드래그 앤 드롭)
  - 업로드된 파일 목록 표시
  - 파일 처리 상태 메시지
- 프로젝트 설명 입력란

**목적:**
- 고객이 만들고 싶은 서비스 유형을 선택하면 AI가 해당 도메인 맥락을 미리 파악
- 기획서/참고자료를 업로드하면 AI가 텍스트를 자동 추출하여 분석에 활용

---

### Step 2. AI 채팅 + 요구사항 추출

**화면 구성:**
- 좌측(또는 상단): AI 채팅 인터페이스
  - 메시지 버블 (사용자 / AI / 시스템)
  - 하단 메시지 입력창
  - AI 타이핑 인디케이터
- 우측(또는 하단): 요구사항 패널
  - 카테고리별 요구사항 목록
  - 각 요구사항의 우선순위/상태 표시
  - 추가/수정/삭제 버튼
- 모바일: 탭 레이아웃으로 채팅/요구사항 전환

**목적:**
- AI가 고객과 대화하며 프로젝트 요구사항을 자연스럽게 수집
- 대화 내용을 실시간으로 구조화된 요구사항으로 추출
- 고객이 추출된 요구사항을 바로 확인하고 수정 가능

---

### Step 3. 프로젝트 개요 생성

**화면 구성:**
- 프로젝트 개요 패널 (ProjectOverviewPanel)
  - 서비스 핵심 요소 정리
  - 주요 기능 목록
  - 타겟 사용자 정의
  - 예상 개발 기간

**목적:**
- 채팅 내용을 정형화된 프로젝트 개요로 변환
- 고객이 "AI가 내 프로젝트를 제대로 이해했는지" 확인하는 단계

---

### Step 4~5. 요구사항 검증 + 결과 확인

**화면 구성:**
- AI 검증 로딩 화면 (AIVerificationLoading)
- 검증 결과 모달 (VerificationResultModal)
- 요구사항 결과 패널 (RequirementsResultPanel)
  - 카테고리별 분류 결과
  - 전체 요구사항 수 표시

**목적:**
- AI가 추출된 요구사항의 완전성/일관성을 자동 검증
- 누락된 영역이나 모호한 항목을 고객에게 피드백

---

### Step 6. 최종 확인 및 제출

**화면 구성:**
- 확인 패널 (ConfirmationPanel)
  - 프로젝트 개요 + 요구사항 + 견적 통합 요약
- 최종 확인 모달 (FinalConfirmationModal)
  - 저장/취소 버튼

**목적:**
- 모든 정보를 한눈에 최종 검토
- 고객의 명시적 승인 후 프로젝트 데이터를 DB에 영구 저장

---

### 부가 기능

| 기능 | 설명 |

## 3. 로그인 (`/auth/login`)

**화면 구성:**
- 중앙 정렬 카드 레이아웃
- 카드 헤더: `계정에 로그인` / `이메일과 비밀번호를 입력해주세요`
- 입력 폼
  - 이메일 입력란
  - 비밀번호 입력란
- 에러 알림 (빨간 배경, 조건부 표시)
- `로그인` 버튼 (인디고, 전체 너비)
  - 로딩 시: `로그인 중...` + 스피너
- 하단: `회원가입` 링크

**목적:**
- 기존 고객이 빠르게 로그인하여 프로젝트에 접근
- 로그인 전 작업 중이던 단계로 자동 복귀 (`redirectStep` 파라미터)

## 4. 회원가입 (`/auth/signup`)

### 가입 폼 상태

**화면 구성:**
- 중앙 정렬 카드 레이아웃
- 카드 헤더: `새 계정 만들기` / `정보를 입력하여 계정을 생성해주세요`
- 입력 폼
  - 이름 (placeholder: `홍길동`)
  - 이메일
  - 회사명 (선택, placeholder: `회사명`)
  - 비밀번호 (6자 이상)
  - 비밀번호 확인
- 각 필드 아래 에러 메시지 (빨간 텍스트)
- `회원가입` 버튼 (인디고, 전체 너비)
  - 로딩 시: `회원가입 중...`
- 하단: `로그인` 링크

### 가입 성공 상태

**화면 구성:**
- 초록색 체크 아이콘
- `회원가입 완료!` 제목
- 파란 안내 박스: `이메일 확인이 필요합니다`
  - 다음 단계 안내 (1~4단계 번호 목록)
- 노란 경고 박스: `이메일이 보이지 않나요? 스팸/광고 편지함도 확인해보세요.`
- `로그인 페이지로 이동` 버튼

**목적:**
- 고객 온보딩 및 B2B 정보 수집 (이름, 회사명)
- 이메일 인증으로 유효한 연락처 확보
- 성공 화면에서 다음 행동을 명확히 안내하여 이탈 방지

## 5. 마이페이지 — 대시보드 (`/mypage`)

### 사이드바 (공통)

**화면 구성:**
- 사용자 프로필 영역: 이니셜 아바타 + 이름 + `프로젝트 매니저`
- 네비게이션 메뉴
  - 🏠 프로젝트 대시보드
  - 📄 진행중인 프로젝트
  - ✅ 완료된 프로젝트
  - ⚙️ 설정
- 현재 페이지 활성 표시 (인디고 배경)
- 모바일: 햄버거 메뉴로 토글

### 대시보드 본문

**화면 구성:**
- 페이지 제목: `프로젝트 대시보드` / `프로젝트 현황을 한눈에 확인하세요`
- 통계 카드 4개 (가로 배열)

| 카드 | 아이콘 | 색상 |
|------|--------|------|
| 진행중인 프로젝트 | ▶️ | 파랑 |
| 완료된 프로젝트 | ✅ | 초록 |
| 총 견적금액 | 💰 | 보라 |
| 승인 대기 | ⏰ | 주황 |

- `최근 프로젝트` 섹션 (최대 6개, 그리드)
  - 프로젝트 카드마다:
    - 제목 + 상태 뱃지
    - 설명 (2줄 제한)
    - 프로젝트 ID · 요구사항 수 · 견적금액 · 수정일
    - 액션 버튼: `견적서` (완료 시) / `이어하기` 또는 `상세보기`
  - 6개 초과 시 `전체 보기` 링크
- 프로젝트 없을 때: `프로젝트가 없습니다` + `새 프로젝트를 시작해보세요`

**목적:**
- 고객이 자신의 전체 프로젝트 현황을 한눈에 파악
- 통계 카드로 포트폴리오 규모 즉시 확인
- 견적서 PDF 다운로드로 내부 보고/예산 승인에 활용
- 이어하기 기능으로 미완료 프로젝트 복구

---

## 6. 마이페이지 — 진행중인 프로젝트 (`/mypage/in-progress`)

**화면 구성:**
- 페이지 제목: `진행중인 프로젝트` / `현재 진행 중인 프로젝트들을 확인하세요 (N개)`
- 필터 영역
  - 상태 필터: `모든 상태` / `요구사항 검토` / `요구사항 추출` / `견적 산출` / `계약 진행` / `진행중`
  - 정렬: `최신순` / `오래된순`
- 프로젝트 카드 그리드 (1~3열 반응형)
  - 제목 + 상태 뱃지 (상태별 색상)
    - 진행중: 파랑 / 요구사항 검토: 노랑 / 요구사항 추출: 주황 / 견적 산출: 보라 / 계약 진행: 인디고
  - 설명 (2줄 제한)
  - 프로젝트 ID · 요구사항 수 · 수정일
  - `작성 이어하기` 버튼
- 빈 상태: `진행중인 프로젝트가 없습니다` + `새 프로젝트 시작하기` 버튼

**목적:**
- 현재 활성 프로젝트만 모아 고객이 "지금 해야 할 일"에 집중
- 세부 상태 필터로 어떤 단계에서 멈췄는지 파악
- 이어하기 기능으로 중도 이탈 프로젝트 완료 유도

---

## 7. 마이페이지 — 완료된 프로젝트 (`/mypage/completed`)

**화면 구성:**
- 페이지 제목: `완료된 프로젝트` / `완료된 프로젝트들을 확인하세요 (N개)`
- 정렬: `최신순` / `오래된순`
- 프로젝트 카드 그리드
  - 제목 + `완료` 뱃지 (초록)
  - 설명
  - 프로젝트 ID · 요구사항 수 · 견적금액 (원) · 완료일
  - 액션 버튼:
    - `견적서 다운로드` (다운로드 아이콘)
    - `상세보기`
- 빈 상태: `완료된 프로젝트가 없습니다` + `새 프로젝트 시작하기` 버튼

**견적서 PDF 내용:**
- 기본 견적 산출 내역
- 프로젝트 단계 (기획/개발/테스트/배포)
- 결제 일정
- 팀 구성
- 요구사항 요약 (필수/권장/선택)

**목적:**
- 완료된 프로젝트의 산출물을 영구 보관/재참조
- 견적서를 언제든 재다운로드 가능
- 과거 프로젝트를 레퍼런스로 활용

---

## 8. 마이페이지 — 설정 (`/mypage/settings`)

**화면 구성:**
- 페이지 제목: `설정` / `계정 및 프로젝트 설정을 관리하세요`
- 상단 메시지 알림 (성공: 초록 / 에러: 빨강, 조건부)

### Notion 연동 카드

**연결 안됨 상태:**
- 회색 상태 박스
  - 회색 상태 점 + `연결 안됨`
  - `Notion 계정을 연결하여 문서를 공유하세요.`
  - `Notion 계정 연결하기` 버튼

**연결됨 상태:**
- 초록 상태 박스
  - 초록 상태 점 + `연결됨`
  - 워크스페이스 이름 · 연결일 표시
  - `연결 해제` 버튼
- 파란 안내 박스: `Notion 데이터베이스 ID`
  - 설정 방법 안내 (번호 목록)
  - 데이터베이스 ID 입력란
  - `저장` 버튼
  - 미설정 시 경고 메시지

**목적:**
- 고객이 사용 중인 Notion 워크스페이스와 연동
- 요구사항/견적서를 Notion에 자동 공유하여 팀 협업 지원
- 연결/해제를 고객이 직접 관리 (데이터 주권)

## 9. 관리자 — 대시보드 (`/admin`)

### 관리자 사이드바 (공통)

**화면 구성:**
- 로고: `F` 원형 + `FLOWGENCE` 텍스트
- 사용자 프로필: 이니셜 아바타 + 이름 + `관리자`
- 네비게이션 메뉴
  - 🏠 대시보드
  - 👥 사용자 관리
  - 📊 프로젝트 모니터링
  - 📄 견적/계약 (비활성)
  - ⚙️ 설정 (비활성)
- 데스크톱 상단 바: `마이페이지` · `문의하기` 링크

### 대시보드 본문

**화면 구성:**
- 검색바: `검색...`
- 통계 카드 4개 (가로 배열)

| 카드 | 아이콘 | 색상 |
|------|--------|------|
| 진행중인 프로젝트 | ▶️ | 파랑 |
| 완료된 프로젝트 | ✅ | 초록 |
| 총 견적금액 | 💰 | 보라 |
| 승인 대기 | ⏰ | 주황 |

- 2열 레이아웃 (데스크톱)

**좌측: 최근 프로젝트**
- `전체보기 →` 링크
- 프로젝트 항목: 제목 + 상태 뱃지 + 사용자명 + 생성일

**우측: 승인 대기 검토**
- 프로젝트 항목: 제목 + `상세보기` 버튼 + 견적금액 + 대기 일수 + 사용자명

### 프로젝트 상세 모달

**화면 구성:**
- 오버레이 + 모달 헤더: `프로젝트 상세` + 닫기 버튼
- 스크롤 가능한 본문:
  - **기본 정보**: 프로젝트명 · 사용자명 · 상태 뱃지 · 요청일 · 수정일 · 설명
  - **프로젝트 개요**: 서비스명 · 서비스 설명 · 주요 기능 목록 · 타겟 사용자 · 예상 개발 기간
  - **견적 정보**: 총 견적금액 (큰 글씨, 인디고) · 요구사항 수
  - **요구사항 상세**: RequirementsResultPanel 컴포넌트

**목적:**
- 서비스 전체 KPI를 실시간 모니터링
- 신규 프로젝트를 빠르게 확인하여 대응
- 승인 대기 일수로 SLA 위반 방지
- 상세 모달에서 프로젝트 심층 검토

## 10. 관리자 — 사용자 관리 (`/admin/users`)

**화면 구성:**
- 페이지 제목: `사용자 관리`
- 상단 컨트롤
  - 검색: `이메일 또는 이름으로 검색` (돋보기 아이콘)

## 12. 문의하기 (`/contact`)

**화면 구성:**
- 페이지 제목: `문의하기`
- 2열 레이아웃 (모바일은 1열)

### 좌측: 연락처 정보

- 섹션 제목: `연락처 정보`
- 이메일: contact@flowgence.ai
- 주소: 대전광역시 중구 중앙로 101, 대전창업허브 406호

### 우측: 빠른 문의 폼

- 섹션 제목: `빠른 문의`
- 입력 폼
  - 이름 (필수, 2~50자)
  - 이메일 (필수, 유효한 형식)
  - 문의내용 (필수, 10~2,000자, 텍스트에리어)
- 필드별 에러 메시지 (빨간 텍스트)
- 알림 박스 (조건부): 에러(빨강) / 성공(초록)
- `문의 전송` 버튼 (파랑, 전체 너비)
  - 로딩 시: `전송 중...`
- 성공 시 폼 자동 리셋

**목적:**
- AI 자동화로 해결되지 않는 문의를 인간 대응 채널로 전달
- 미가입 잠재 고객도 문의 가능 (리드 확보)
- 이메일(Nodemailer)로 운영팀에 즉시 전달
