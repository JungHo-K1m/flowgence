## Changelog - 랜딩 페이지 메인 영역 중앙 정렬

### 날짜
2025-11-13

### 배경
- 초기 랜딩 화면이 상단에 고정되어 있어 큰 화면에서 여백이 많이 생김
- 사용자 요청: **"메인 페이지 영역을 화면의 중앙(세로방향)에 반응형으로 배치"**

### 변경 사항
1. `page.tsx`( `frontend/src/app/page.tsx` )
   - 랜딩 영역 컨테이너에 `flex-1`, `min-h-[calc(100vh-96px)]` 등을 적용하여 세로 중심 정렬
   - 모바일에서 상단이 잘리지 않도록 `pt-24`(sm 이하)와 반응형 타이포/여백(`text-3xl`, `sm:text-[40px]`, `md:text-[48px]`) 조정
   - 최근 작업/업로드 상태가 존재하면 `md` 구간에서는 `justify-start`, `xl` 이상에서는 `justify-center`를 적용해 작은 화면은 상단 정렬, 큰 화면은 중앙 정렬을 유지

### 영향 범위
- 초기 랜딩 화면에서만 레이아웃 변경
- 이후 단계(요구사항, 확인, 결과)는 기존 레이아웃 그대로 유지

### 테스트 시나리오
1. 데스크톱(>=1024px)에서 랜딩 페이지 진입 → 텍스트/입력 영역이 화면 세로 중앙에 위치하는지 확인
2. 태블릿/모바일에서 화면 높이가 작을 때 → 콘텐츠가 중앙 근처에 위치하되, 스크롤이 필요한 경우 자연스럽게 동작하는지 확인
