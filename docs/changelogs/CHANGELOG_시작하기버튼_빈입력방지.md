## Changelog - 시작하기 버튼 빈 입력 방지

### 날짜
2025-11-13

### 배경
- 랜딩 페이지에서 `시작하기` 버튼을 입력 없이 눌러도 플로우가 진행되어 빈 프로젝트 설명이 전달됨
- 사용자 요청: **"텍스트입력이 아무것도 없을 때는 클릭 동작을 하지 않도록 한다."**

### 변경 사항
1. `page.tsx`( `frontend/src/app/page.tsx` )
   - `userComment`가 공백일 때 버튼을 비활성화하는 `isStartDisabled` 파생 상태 추가
   - 비활성화 시 `onClick` 조기 반환 및 시각적 피드백(`opacity-60`, `cursor-not-allowed`) 적용

### 영향 범위
- 랜딩 페이지에서 직접 `시작하기` 버튼을 누를 때만 동작 제약
- URL 파라미터나 자동 시작 로직(`autoStart`)은 기존 그대로 유지

### 테스트 시나리오
1. 텍스트 입력이 비어 있는 상태에서 `시작하기` 버튼 클릭 → 아무 일도 일어나지 않는지 확인
2. 텍스트 입력 후 `시작하기` 버튼 클릭 → 채팅 인터페이스로 정상 전환되는지 확인
